# MCP 2026-07-28 spec upgrade (#424)

## Problem

MCP 2026-07-28 removes the `initialize`/`initialized` handshake and `Mcp-Session-Id` session model, adds mandatory `Mcp-Method`/`Mcp-Name` routing headers, introduces `server/discover` for capability negotiation, and adds `ttlMs`/`cacheScope` caching metadata on list/read responses. KiwiFS used `mcp-go` Streamable HTTP with `WithStateLess(true)` but lacked the new protocol surface — clients negotiating 2026-07-28 could not discover capabilities or cache tool lists.

## Root cause

`mark3labs/mcp-go` v0.49–v0.55 implements through 2025-11-25 only (`LATEST_PROTOCOL_VERSION`). No upstream support yet for `server/discover`, response routing headers, or list caching fields. KiwiFS delegated entirely to `NewStreamableHTTPServer` without a compatibility wrapper.

## Solution

Added `internal/mcpserver/mcp2026.go` — a Streamable HTTP transport wrapper (`wrapMCP2026`) around the existing handler:

1. **`server/discover`** — handled before mcp-go; builds response via internal `initialize` to reuse capability assembly. Returns `supportedVersions`, `capabilities`, `serverInfo`, `instructions`, `ttlMs` (1h), `cacheScope` (`public`).
2. **`Mcp-Method` / `Mcp-Name` headers** — emitted on all JSON responses; validated on requests when client declares protocol `2026-07-28` (via `Mcp-Protocol-Version` header or `_meta.io.modelcontextprotocol/protocolVersion`). `Mcp-Name` required only for `tools/call`, `resources/read`, `prompts/get` (SEP-2243).
3. **`ttlMs` / `cacheScope`** — injected into `tools/list`, `resources/list`, `resources/templates/list`, `prompts/list`, and `resources/read` JSON responses (5 min TTL, `public` scope) without overwriting server-provided values.
4. **Backward compatibility** — legacy clients using `initialize` with 2024-11-05 / 2025-11-25 and no routing headers continue to work. Stateless mode (`WithStateLess(true)`) unchanged — no `Mcp-Session-Id` returned.
5. **JSON Schema 2020-12** — `tools/list` responses get `$schema: https://json-schema.org/draft/2020-12/schema` on each tool `inputSchema` when absent. Already supported by `mcp-go` + `google/jsonschema-go` for schema features.

`StreamableHTTPHandler` now returns `bearerAuth(authToken, wrapMCP2026(mcpHandler, s))`.

## Peer review notes

- When `mcp-go` adds native 2026-07-28 support, evaluate removing the wrapper and migrating to upstream.
- OAuth 2.1 PKCE and MCP Tasks extension are separate follow-ups (issue should-have / nice-to-have).
- SSE upgrade responses skip JSON caching enhancement (content-type `text/event-stream`).

## Reuse guide

- To adjust list TTL: change `defaultListTTLMs` / `defaultDiscoverTTL` in `mcp2026.go`.
- To add a new list method with caching: extend `enhanceListCaching` switch.
- For 2026 clients: send `Mcp-Protocol-Version: 2026-07-28`, `Mcp-Method` matching JSON-RPC method, and `Mcp-Name` for named calls.
- Search Kiwi: `kiwi_search "MCP 2026-07-28"` or `issue-424`.
