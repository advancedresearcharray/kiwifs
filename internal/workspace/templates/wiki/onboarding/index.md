# Onboarding

Everything a new team member needs in their first two weeks.

## Before Day 1

- [ ] Laptop / hardware ordered
- [ ] Accounts created (email, Slack, GitHub, cloud provider)
- [ ] Added to relevant Slack channels
- [ ] Calendar invites sent for recurring meetings
- [ ] Buddy/mentor assigned

## Week 1 — Get Set Up

- [ ] Read [[welcome]] to understand how this wiki works
- [ ] Read [[how-we-work]] for team norms and meeting rhythms
- [ ] Set up local development environment (see [[processes/dev-setup|Dev Setup]])
- [ ] Clone repos and verify you can build and run locally
- [ ] Read [[architecture]] for system overview
- [ ] Meet your buddy/mentor for a walkthrough
- [ ] Join your first standup

## Week 2 — Get Productive

- [ ] Pick up a starter task (look for `good-first-issue` in the tracker)
- [ ] Open your first PR and get it reviewed
- [ ] Read through [[decisions/index|recent ADRs]] to understand past choices
- [ ] Read the [[reference/glossary|Glossary]] for team-specific terms
- [ ] Attend your first planning and retro meeting
- [ ] Ask at least 3 questions — no question is too basic
